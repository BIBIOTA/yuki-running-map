## ADDED Requirements

### Requirement: admin_units stores Taiwan county and township polygons

The system SHALL add an `admin_units` table via migration `0006_add_admin_units.sql` with columns `id uuid PRIMARY KEY`, `code text UNIQUE NOT NULL` (тЁДТћ┐жЃе administrative code), `level admin_level NOT NULL` (enum `county | township`), `name text NOT NULL`, `parent_code text REFERENCES admin_units(code) DEFERRABLE INITIALLY DEFERRED` (county-level rows have NULL parent_code; township-level rows reference their parent county), and `geom geometry(MultiPolygon, 4326) NOT NULL`. The migration SHALL create a `GIST(geom)` index for spatial queries and a `btree(level)` index for level filtering. The table SHALL be readable by anonymous users (RLS `anon SELECT` permitted) since it is a public reference dataset. Admin write access is provided only through the migration itself (no admin write policy required at runtime).

#### Scenario: Migration creates table and indexes
- **WHEN** `pnpm db:migrate` runs `0006_add_admin_units.sql`
- **THEN** the `admin_units` table exists with columns matching the specification
- **AND** the `admin_level` enum type exists with values `county` and `township`
- **AND** indexes `admin_units_geom_gist` and `admin_units_level_idx` exist
- **AND** RLS is enabled on `admin_units` and `anon SELECT` is permitted

> See: ../../designs/figma.md

### Requirement: seed migration imports Taiwan admin units from GeoJSON

The system SHALL seed `admin_units` via migration `0007_seed_taiwan_admin_units.sql` using a pre-processed `lib/db/migrations/seed/taiwan-admin-units.geojson` file committed to the repository. The seed file SHALL be generated by `scripts/build-admin-units-geojson.ts` from тЁДТћ┐жЃе SHP source data (this script SHALL be committed for future year-on-year refresh per design.md ┬Д10.5 runbook). The generator script SHALL normalise each feature to include `properties.code` (тЁДТћ┐жЃе administrative code), `properties.level` (`county` or `township`), `properties.name`, and `properties.parent_code` (NULL for counties). The generator SHALL apply `ST_MakeValid` to repair any self-intersecting polygons before output. After the migration runs, the table SHALL contain exactly 22 county rows and approximately 370 township rows, and every township SHALL reference a valid county code via `parent_code`.

#### Scenario: Seed migration loads counties and townships
- **WHEN** `pnpm db:migrate` runs `0007_seed_taiwan_admin_units.sql` (after `0006`)
- **THEN** `SELECT COUNT(*) FROM admin_units WHERE level = 'county'` returns 22
- **AND** `SELECT COUNT(*) FROM admin_units WHERE level = 'township'` returns a value between 360 and 380 (rounded to the source SHP era)
- **AND** every county row has `parent_code IS NULL`
- **AND** every township row's `parent_code` exists in `admin_units.code` at level `county`
- **AND** every `geom` is a valid `MultiPolygon` (validated via `ST_IsValid(geom)`)

> See: ../../designs/figma.md

### Requirement: route_admin_units join table associates routes with admin_units

The system SHALL add a `route_admin_units` join table via migration `0006_add_admin_units.sql` with primary key `(route_id, admin_unit_id)`, foreign key `route_id REFERENCES routes(id) ON DELETE CASCADE`, foreign key `admin_unit_id REFERENCES admin_units(id) ON DELETE RESTRICT`, and a `btree(admin_unit_id)` index for reverse lookups (which routes intersect a given admin unit). RLS SHALL be enabled on the table and SHALL mirror the routes-table policy: anonymous SELECT is allowed only for rows whose `route_id` corresponds to a `published = true` row in `routes`; admin full access is governed by the existing `app_admin_github_username()` function.

#### Scenario: Migration creates join table with correct constraints
- **WHEN** `pnpm db:migrate` runs `0006_add_admin_units.sql`
- **THEN** the `route_admin_units` table exists with PK `(route_id, admin_unit_id)`
- **AND** the `route_id` FK has `ON DELETE CASCADE`
- **AND** the `admin_unit_id` FK has `ON DELETE RESTRICT`
- **AND** the index `route_admin_units_admin_unit_idx` exists on `(admin_unit_id)`
- **AND** RLS policies match the spec (anon SELECT gated by routes.published; admin full access)

> See: ../../designs/figma.md

#### Scenario: Deleting a route cascades the join rows
- **WHEN** a row in `routes` is deleted (e.g., via the `deleteRoute` Server Action from the admin-routes-crud capability)
- **THEN** every `route_admin_units` row whose `route_id` matches is also deleted
- **AND** the corresponding `admin_units` rows are NOT deleted (RESTRICT on the admin_unit_id FK)

### Requirement: 0008 backfills route_admin_units and drops legacy region column

The system SHALL provide migration `0008_drop_routes_region_add_backfill.sql` that (1) backfills `route_admin_units` from existing routes by running `INSERT INTO route_admin_units (route_id, admin_unit_id) SELECT r.id, a.id FROM routes r, admin_units a WHERE ST_Intersects(a.geom, ST_GeomFromGeoJSON(r.geojson->'geometry'))` filtered by `WHERE NOT EXISTS (SELECT 1 FROM route_admin_units rau WHERE rau.route_id = r.id)` (so prior partial state is not re-inserted), and (2) drops the legacy `routes.region` text column. The migration SHALL NOT abort if individual routes fail spatial intersection (e.g., offshore routes match zero polygons Рђћ that is permitted state); failures of the join INSERT SHALL be logged but not block the column drop.

#### Scenario: Backfill populates join table for existing routes
- **WHEN** `pnpm db:migrate` runs `0008_drop_routes_region_add_backfill.sql` against a DB that already has routes seeded
- **THEN** every route whose `geojson` LineString intersects at least one `admin_units.geom` polygon has one or more corresponding `route_admin_units` rows
- **AND** routes with zero intersecting admin units result in zero new `route_admin_units` rows (no error)
- **AND** `routes` no longer has a `region` column after the migration completes

> See: ../../designs/figma.md

### Requirement: detectRegions performs the spatial query helper

The system SHALL expose `detectRegions(tx, geojsonGeometry): Promise<string[]>` from `lib/admin-routes/detectRegions.ts`. The function SHALL execute `SELECT id FROM admin_units WHERE ST_Intersects(geom, ST_GeomFromGeoJSON($1))` against the provided transaction and return the resulting `admin_unit_id` array. The input geometry SHALL be the simplified `LineString` from `parseGpx(buffer).geojson.geometry` (NOT raw trackpoints Рђћ the simplified form is sufficient for intersection detection and avoids large payloads). The function SHALL return `[]` (empty array) when no admin units intersect, without throwing. Database errors (malformed geometry, transaction state issues) SHALL propagate so the caller (`createRoute`) can roll back the surrounding transaction.

#### Scenario: Line crossing two townships returns two admin_unit_ids
- **WHEN** `detectRegions(tx, geojsonLine)` is called with a LineString that crosses two `admin_units` polygons (e.g., a Taipei loop crossing СИГТГБтЇђ and тцДт«ЅтЇђ)
- **THEN** the result is an array of exactly two `admin_unit_id` values
- **AND** each value matches an `admin_units.id` whose `geom` actually intersects the input line

> See: ../../designs/figma.md

#### Scenario: Offshore line returns empty array
- **WHEN** `detectRegions(tx, geojsonLine)` is called with a LineString entirely over open sea (no land polygon intersected)
- **THEN** the result is exactly `[]`
- **AND** no exception is thrown

#### Scenario: Edge-grazing line still counts as intersection
- **WHEN** `detectRegions(tx, geojsonLine)` is called with a LineString whose only contact with a polygon is a single shared boundary point (`ST_Intersects` returns `true` for shared boundaries)
- **THEN** the result includes that polygon's `admin_unit_id`

### Requirement: RouteRegions renders per-county text paragraphs across surfaces

The system SHALL provide a Server Component `<RouteRegions regions={Region[]} variant?: 'stacked' | 'inline' />` under `components/RouteRegions.tsx`, plus the supporting type `Region = { code: string; level: 'county' | 'township'; name: string; parent_code?: string | null }` in `lib/regions/types.ts`. When `regions` is empty the component SHALL return `null` (rendering nothing, including no wrapping element). The default `'stacked'` variant SHALL group regions by county (`level === 'county'` plus its `township` children, identified via `parent_code === county.code`) and render one paragraph per county with format `{уИБтИѓ} Рђћ {жёЅжј«1сђЂжёЅжј«2Рђд}`. Within each paragraph the county name SHALL use Inter Medium + forest-green colour (`text-primary`) and the township names SHALL use Inter Regular + ink colour. Multiple counties SHALL stack vertically. The `'inline'` variant SHALL render a single-line composite `{уИБтИѓ} {жёЅжј«Рђд} / {уИБтИѓ} {жёЅжј«Рђд}` with CSS truncation (`white-space:nowrap; overflow:hidden; text-overflow:ellipsis`) for use in table cells (e.g., admin routes list `тЇђтЪЪ` column). The component SHALL NOT render chip / badge / pill UI elements Рђћ it is paragraph-style only (per Figma frame `70:9` and design.md ┬Д2.3 decision after iteration with the user).

#### Scenario: Empty regions renders nothing
- **WHEN** `<RouteRegions regions={[]} />` is rendered
- **THEN** the output is `null` (no DOM element)
- **AND** no wrapping `<div>`, `<ul>`, or text node is emitted
