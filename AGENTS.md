# AGENTS.md

Conventions for any contributor (human or AI) editing **Yuki's Running Map**.

If you skip this doc you will trip over the language policy, the folder boundaries, the testing expectations, or the commit format — and reviewers will ask you to redo it. Take five minutes here.

## Language policy

- **Conversation / explanations / PR descriptions / Markdown prose**: 繁體中文 by default. Yuki is the primary stakeholder.
- **Code identifiers / file names / commit messages / OpenSpec keywords (`ADDED`, `MODIFIED`, `REMOVED`, `WHEN`, `THEN`, `AND`, `SHALL`)**: English.
- **Code comments**: prefer English so contributors who arrive later (and machine translators) can read them without context switching. Domain terms like 「淡水河」 / 「LSD」 can stay 中文 inside UI strings.

## Tech stack

The full diagram is in [docs/architecture.md](./docs/architecture.md). One-liner:

> **Next.js 15 App Router + React 19** on **Vercel**, talking to **Supabase** (Postgres + PostGIS + Storage + Auth/GitHub OAuth), with **MapLibre GL JS** rendering **Protomaps PMTiles** tiles hosted on Supabase Storage. UI styled with **Tailwind v4** + **shadcn/ui** primitives + V2 Trail Vintage tokens.

ORM is **Drizzle**, fonts are loaded via `next/font/google` (Fraunces / Inter / IBM Plex Mono), icons via **lucide-react**.

## Code style

- **TypeScript strict + `noUncheckedIndexedAccess`** — non-negotiable. No `any` (ESLint `no-explicit-any: error`). Prefer `unknown` + narrowing.
- **Type imports**: ESLint enforces `consistent-type-imports` — write `import type { Foo } from "..."`.
- **Server Components by default**. Add `"use client"` only when you actually need browser APIs (state, refs, event handlers, MapLibre).
- **Naming**: `kebab-case.tsx` for route files (Next.js convention), `camelCase.ts` for libraries, `PascalCase.tsx` for components.
- **Imports**: handled by `@ianvs/prettier-plugin-sort-imports`. Don't fight the auto-order.
- **Tailwind classes**: handled by `prettier-plugin-tailwindcss`. Don't reorder by hand.

## Folder boundaries

```
app/                # Next.js App Router (route handlers, layouts, pages)
components/         # UI components
  ui/               # shadcn/ui primitives — owned by the design system, do not hand-edit
features/           # Feature-scoped code (route-list, route-detail, admin-upload, …)
lib/                # Shared, framework-agnostic code
  db/               # Drizzle schema + migrations
  gpx/              # GPX parsing / simplification / metadata
  map/              # MapLibre + PMTiles helpers
  supabase/         # Supabase client factories (browser / server / middleware)
  utils.ts          # cn() and other tiny helpers
docs/               # Long-form architectural docs + runbooks
public/             # Static assets (Logo, favicon, OG images)
openspec/           # OpenSpec changes + capability specs
middleware.ts       # Edge middleware (admin route guard)
```

**Rule**: code under `features/<A>` MUST NOT import from `features/<B>`. If two features need the same thing, hoist it to `lib/`. This keeps each feature replaceable without a giant ripple.

## Testing

| Layer       | Tool                    | Scope                                                                                                                                                                               |
| ----------- | ----------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Unit        | Vitest                  | `lib/gpx`, `lib/map`, Drizzle schema validators                                                                                                                                     |
| Integration | Vitest + Supabase local | RLS policy behaviour, PostGIS query results                                                                                                                                         |
| E2E         | Playwright              | The five user-visible flows (visitor browses list, visitor reads detail + downloads GPX, admin logs in, admin uploads GPX, unauthenticated user is redirected from `/admin/upload`) |

Bootstrap is shipping `lint` + `typecheck` only — Vitest + Playwright wire up in Waves B / C as covered by tasks 5.2 and 8.3.

## Commits

Use **Conventional Commits**. Examples:

```
feat(routes): add GPX upload form
fix(middleware): handle missing ADMIN_GITHUB_USERNAME env
docs(runbooks): refresh PMTiles bbox to include Hokkaido
chore(deps): bump next 15.5 → 15.6
```

Co-author trailer is added automatically when a commit is generated by Claude Code.

## PRs ↔ OpenSpec changes

**Every PR maps to exactly one `openspec/changes/{change-id}/`.** A PR opens by `brainstorming` and closes by `verification-before-completion`. Don't bundle two changes into one PR — split them.

The PR description should link to:

- `openspec/changes/{change-id}/proposal.md`
- The relevant `specs/{capability}/spec.md` requirements
- Any `diagrams/*.puml` or `designs/figma.md` artifacts referenced by the spec

## Environment variables

`.env.local` is git-ignored. Required vars (kept in sync with `.env.example`):

| Var                             | Where used                               | How to obtain                                                                                                                             |
| ------------------------------- | ---------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| `NEXT_PUBLIC_SUPABASE_URL`      | client + server                          | Supabase dashboard → Project settings → API                                                                                               |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | client + server                          | same                                                                                                                                      |
| `SUPABASE_SERVICE_ROLE_KEY`     | server-only (Server Actions, migrations) | same — **NEVER expose to client**                                                                                                         |
| `ADMIN_GITHUB_USERNAME`         | middleware                               | Yuki's GitHub username (e.g. `bibiota`)                                                                                                   |
| `NEXT_PUBLIC_PMTILES_URL`       | client (MapLibre)                        | Supabase Storage public URL to the `tiles/<name>.pmtiles` file (see [docs/runbooks/pmtiles-update.md](./docs/runbooks/pmtiles-update.md)) |

## Don't

Repeats from [CLAUDE.md](./CLAUDE.md):

- Don't introduce new deps without asking.
- Don't import across `features/*` boundaries.
- Don't edit `next-env.d.ts`.
- Don't commit `.env*.local` or `.next/`.
- Don't rewrite design tokens without going through `spec-driven-dev:updating-spec`.

## When in doubt

1. Check [docs/architecture.md](./docs/architecture.md) for runtime / boundary questions.
